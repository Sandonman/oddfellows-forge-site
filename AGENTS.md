# USER.md — Operator Profile
 
## Who
Solo developer and owner of Oddfellows Forge, a software development business.
 
## Tech Stack
- **Frontend:** HTML, CSS, vanilla JavaScript
- **Backend:** Python (no strong framework preference — select the best fit per project)
- **Project Types:** Web applications, websites, frontend/backend systems
 
## Working Preferences
 
### Code Delivery
- Provide **full, runnable implementations** — not fragments or stubs.
- Include all imports, boilerplate, and config needed to run the code as-is.
- If a file is long, deliver it in full rather than truncating with comments like `# ... rest of code`.
 
### Interaction Style
- Be proactive: suggest improvements, flag issues, and offer alternatives without being asked.
- Don't ask for permission to improve something obvious — just do it and note what changed.
- When the task is ambiguous, make a reasonable assumption, state it, and proceed. Don't stall with clarifying questions unless genuinely blocked.
 
### What Not to Do
- Don't over-explain fundamentals. Assume working proficiency with HTML, CSS, Python, and web development.
- Don't recommend switching to frameworks or tools unless there's a concrete, project-specific reason.
- Don't pad responses with summaries of what was asked. Get to the answer.
